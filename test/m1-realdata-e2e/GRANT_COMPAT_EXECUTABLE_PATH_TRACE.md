## Ordering, state, and observations

1. **Generate live material before node startup.** The pinned g-06b generator's
   `--at-instant` and `--output-dir` arguments are implemented at
   `test-vectors/grant-output/generate.py:289-304`; a non-default instant is
   refused unless output is outside the frozen directory. The test invokes the
   vendored script in a temporary directory. Observation: all five generated
   files parse, and every plane-2 node case has the same case identity and
   normalized claims as its frozen twin after removing only time-derived fields,
   signatures, CIDs, and their derived serialization fields. A second run at the
   default instant must reproduce every frozen file byte-for-byte. These are
   behavioral generator observations, not artifact-existence claims.
2. **Start and provision the real node.** `tinycloud::app` opens storage, runs
   migrations through `TinyCloud::new`, constructs `SqlService`, and mounts and
   manages `/delegate` and `/invoke`
   (`tinycloud-node-server/src/lib.rs:110-154,199-217,244-262,296-315`). The w5
   harness establishes the required order: Figment/app, managed SQL service,
   setup-only space row and block directory, then seed data
   (`tests/cross_layer_contract.rs:205-261`). Observation: exactly one space and
   zero delegation/ability rows before imports. Space provisioning is a storage
   prerequisite; no delegation or ability authority is directly inserted.
3. **Import the bounded parent.** Compose the SIWE/CACAO parent from each
   material set's `parentFormatVector`, then submit its exact DAG-CBOR base64url
   through the real `/delegate` Authorization header. The request guard and
   decoder are `tinycloud-node-server/src/authorization.rs:14-38` and
   `tinycloud-auth/src/authorization.rs:27-59`; the route calls
   `TinyCloud::delegate` (`tinycloud-node-server/src/routes/mod.rs:206-276`,
   `tinycloud-core/src/db.rs:507-536`). The processor orders verify, validate,
   save (`tinycloud-core/src/models/delegation.rs:142-180,184-328,415-490`).
   Observation: native success CID equals `parentFormatVector.expectedCid`, and
   the corresponding persisted row exists. Parent import always precedes child
   import except the intended missing-parent case.
4. **Plane 1, frozen-byte oracle and direct imports.** Independently decode the
   frozen UCANs, verify Ed25519 signatures, structure, declared CID identity,
   containment shape, and layer tags. Send unchanged frozen bytes through
   `/delegate` only for invalid-signature (signature verification precedes time),
   invalid-time, and parent-time-nesting cases. Observation: native
   `InvalidSignature` or `InvalidTime` classifications, with the nesting case
   still unmasked under its 2027 parent. Separately import the expired frozen
   ACCEPT token and record native `InvalidTime` as a labeled **LONGEVITY
   OBSERVATION**, never ACCEPT conformance and never the invalid-time case.
5. **Plane 2, live node import.** Submit time-valid generated bytes for every
   ACCEPT case; missing-persisted-parent, terminal-parent, resource/ability and
   caveat containment cases; and every node-invocation case. Parent lookup,
   terminal rejection, time nesting, and containment are implemented at
   `tinycloud-core/src/models/delegation.rs:199-328`. Observation: ACCEPT returns
   the generated declared CID; rejects preserve the case's native
   `MissingParents`, `TerminalParentCannotRedelegate`, `InvalidTime`,
   `UnauthorizedCapability`, or `CaveatsNotContained` classification. An
   observed `InvalidTime` for a differently intended case fails the test.
6. **Enforce through real invocation.** Seed SQL through managed production
   `SqlService`; seed KV using an owner-signed `/invoke`, following the existing
   harness (`tests/cross_layer_contract.rs:1090-1183`). Holder-signed `/invoke`
   passes through the request guard and `invoke_impl`
   (`tinycloud-node-server/src/routes/mod.rs:323-388,688-752`). Native invocation
   validates signature/time, parent lookup, delegatee, chain time, and actions

   `tinycloud-node-server/src/routes/mod.rs:1182-1298,2191-2227`; KV dispatch is

7. **Production-emitter interop.** Construct the actual g-07

## Deliberate node non-enforcements

- The node does not know or compare `grantIssuerDid`; native signature and
  parent authority are the enforcement boundary.
- It does not interpret grant provenance facts except signed
  `delegationMode` (`tinycloud-core/src/util.rs:48-75,97-150` and
  `tinycloud-core/src/models/delegation.rs:415-443`).
- Missing `delegationMode` is attenuable; `terminal` is rejected when later
  cited as a parent (`tinycloud-core/src/models/delegation.rs:220-228,382-393`).
- Holder mismatch may import; native invocation rejects the wrong invoker
  (`tinycloud-core/src/models/invocation.rs:171-178`).

## Unsupported hops and terminal conclusion

The unchanged frozen ACCEPT child expired at `2026-07-10T12:05:00Z`, and the
real route validates ambient time with no seam. Therefore **frozen ACCEPT bytes
-> successful `/delegate` is an unsupported hop**. It is not used or claimed.
Amendment 38 resolves, rather than reinterprets, that limitation: frozen bytes
remain the plane-1 byte/static oracle and longevity observation; generator-made
current-instant bytes supply plane-2 live-path evidence, guarded by normalized
semantic correspondence and default-instant byte identity.

No required acceptance flow needs a mock transport, direct authority-state
mutation, canned evidence, or artifact-existence substituted for behavior. The
ticket is constructible and may proceed. Any newly observed product-path defect
will stop implementation and park the ticket with this trace updated as the
report.
